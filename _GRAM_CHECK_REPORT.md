# GRAM评分系统严格检查报告

## 执行摘要

本报告对黄金量化分析系统的GRAM评分体系进行了严格检查，发现多个严重问题。

---

## 1. 数据真实性问题（严重）

### 1.1 央行购金数据异常

**问题**：
- 测试结果：`central_bank_buying = 97.71吨/月`
- 合理性分析：
  - 全球央行年购金量约1000-1200吨
  - 月均约80-100吨
  - 中国央行占全球20-30%，月均约20-30吨
  - **97.71吨/月异常高**，意味着中国年购金约1172吨，接近全球总量

**根源**：
- `akshare.macro_china_fx_gold()`的`黄金储备-环比`列是**百分比变化率**
- 代码错误地使用 `环比 × 22` 计算购金量
- 22可能是想表示"1%变化=22吨"，但计算逻辑错误

**修复方案**：
```python
# 正确做法：使用"黄金储备-数值"列，计算月度增量
fx_gold = ak.macro_china_fx_gold()
if len(fx_gold) >= 2:
    latest = fx_gold.iloc[-1]['黄金储备-数值']
    prev = fx_gold.iloc[-2]['黄金储备-数值']
    # 假设单位是吨
    monthly_purchase = abs(latest - prev)
```

**当前状态**：已修复为使用`黄金储备-数值`计算月度增量，但单位可能仍有误

## 2. 评分公式问题（中等）

### 2.1 机会成本因子计算

**当前公式**：
```python
oc_avg = (rate_score + dxy_score) / 2.0
```

**问题**：
- 简单平均利率评分和美元评分
- 未考虑历史相关性（利率和美元对黄金的影响程度不同）

**改进方案**：
```python
# 基于历史回归的权重
rate_weight = 0.6  # 利率对黄金影响更大
dxy_weight = 0.4
oc_avg = rate_score * rate_weight + dxy_score * dxy_weight
```

---

### 2.2 趋势动能因子计算

**当前公式**：
```python
momentum = (r1d + r5d + r20d) / 3.0
```

**问题**：
- 简单平均1日、5日、20日收益率
- 未考虑时间衰减（近期收益应权重更高）

**改进方案**：
```python
# 指数加权移动平均
weights = [0.5, 0.3, 0.2]  # 1日, 5日, 20日
momentum = r1d * weights[0] + r5d * weights[1] + r20d * weights[2]
```

## 3. 评分体系合理性问题（严重）

### 3.1 缺少"经济扩张"因子

**WGC GRAM框架**：
1. 经济扩张（Economic Expansion）
2. 风险与不确定性（Risk and Uncertainty）
3. 机会成本（Opportunity Cost）
4. 趋势动能（Momentum/Trend）

**当前实现**：
1. 机会成本（40%）
2. 风险与不确定性（25%）
3. 供需格局（20%）- **不是"经济扩张"**
4. 趋势动能（15%）

**问题**：
- 因子定义与WGC框架不符
- "供需格局"只用央行购金，太片面

**改进方案**：
1. 添加"经济扩张"因子（GDP增长率、工业增加值、消费者信心指数）
2. 扩展"供需格局"因子（央行购金 + ETF持仓 + 珠宝需求 + 工业需求）

---

### 3.2 权重分配主观

**当前权重**：
- 机会成本：40%
- 风险与不确定性：25%
- 供需格局：20%
- 趋势动能：15%

**问题**：
- 权重是否基于历史数据优化？
- 还是主观设定？

**改进方案**：
1. 使用历史数据回测，优化权重
2. 使用IC（信息系数）或ICIR（信息系数稳定性）确定权重
3. 参考WGC官方文档的权重建议

---

### 3.3 评分阈值主观

**当前阈值**：
- 机会成本：国债收益率 < 2.0% = 8分，< 2.5% = 6分，< 3.0% = 4分，< 4.0% = 2分
- 风险与不确定性：VIX > 30 = 9分，> 25 = 7分，> 20 = 5分，> 15 = 3分
- 供需格局：央行购金 > 70吨/月 = 9分，> 50 = 7分，> 30 = 5分
- 趋势动能：动量 > 5% = 9分，> 2% = 7分，> -2% = 5分

**问题**：
- 阈值是否基于历史数据回测？
- 还是拍脑袋定的？

**改进方案**：
1. 使用历史数据，分析各因子与黄金收益率的相关性
2. 根据相关性确定评分阈值
3. 使用分位数（percentile）而非固定阈值
